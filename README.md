# Mini NPU Simulator

## 실행 방법
- Python 3.8 이상에서 실행한다.
- 프로젝트 루트에서 `python main.py` 또는 `python3 main.py`를 실행한다.
- 실행 후 메뉴에서 `1`을 입력하면 3x3 사용자 입력 모드, `2`를 입력하면 `data.json` 분석 모드가 동작한다.
- `data.json`은 프로젝트 루트에 있어야 하며, 없거나 JSON 형식이 잘못되면 프로그램이 친절한 오류 메시지를 출력하고 종료한다.

## 구현 요약
- 행렬은 모두 `list[list[float]]`로 통일해서 저장하고, 입력값이 정수여도 내부에서는 `float`로 처리한다.
- MAC 연산은 외부 라이브러리 없이 이중 반복문으로 직접 구현했다.
- 콘솔 입력 모드에서는 각 줄마다 열 개수와 숫자 파싱을 검증하고, 오류가 나면 같은 줄을 다시 입력받는다.
- JSON 모드에서는 `filters`와 `patterns` 구조를 읽고, `size_{N}_{idx}` 키에서 크기를 추출해 맞는 필터 묶음을 자동 선택한다.
- 표준 라벨은 `Cross`, `X` 두 개만 내부에서 사용하며, `+`, `cross`, `x` 같은 입력값은 모두 이 표준 라벨로 정규화한다.
- 판정은 raw MAC 점수 자체가 아니라, `abs(mac_score - pattern_sum)`으로 계산한 절대오차 기준으로 수행한다. 여기서 `pattern_sum`은 입력 패턴의 전체 합이며, 각 필터의 MAC 결과가 이 기준값에 더 가까운 쪽을 정답으로 본다.
- 동점 비교는 `abs(distance_cross - distance_x) < 1e-9` 기준의 epsilon 정책을 사용해 `UNDECIDED`로 처리한다.
- JSON 모드의 실패는 예외로 프로그램을 중단하지 않고, 케이스 단위 `FAIL` 결과와 사유로 남긴다.
- 성능 분석은 `time.perf_counter()`로 MAC 함수 호출 구간만 10회 반복 측정해서 평균 ms를 출력한다.

## 결과 리포트
- 기본 제공한 `data.json` 샘플 기준으로 총 12개 케이스를 분석하도록 구성했다.
- 이 샘플에서는 8개가 `PASS`, 4개가 `FAIL`이 되며, 실패 케이스는 `size_13_1`, `size_5_4`, `size_5_5`, `size_5_6`이다.
- `size_13_1`은 모든 칸이 1인 패턴이라 Cross와 X 모두 패턴 총합과의 절대오차가 같아진다.
- 두 필터의 절대오차 차가 epsilon보다 작기 때문에 판정은 `UNDECIDED`가 되고, expected 가 `X`라서 `FAIL`이 된다.
- `size_5_3`은 소수점 패턴을 사용한 간발의 차 승부 케이스로, X 쪽 거리가 Cross 쪽 거리보다 약 `2e-9` 더 작아서 epsilon 기준을 넘기며 `PASS`가 된다.
- `size_5_4`는 소수점 패턴을 사용한 epsilon 동점 케이스로, X 쪽 거리가 더 작지만 차이가 약 `5e-10`이라 `UNDECIDED`로 처리되어 `FAIL`이 된다.
- `size_5_5`는 의도적으로 expected 를 `+`로 둔 직접 불일치 케이스라서, 실제 판정은 `X`인데 기대값은 `Cross`가 되어 `prediction_mismatch`가 발생한다.
- `size_5_6`는 의도적인 크기 불일치 케이스로, `size_5` 키를 쓰지만 실제 입력 행렬은 `4x4`라서 `size_mismatch` 실패를 재현한다.
- 이 케이스들 덕분에 현재 데이터셋은 정수 기반 MAC 계산뿐 아니라, 패턴 총합 기준 절대오차 판정 정책과 부동소수점 동점 처리, 스키마/크기 검증이 실제 결과에 어떤 영향을 주는지도 함께 보여준다.
- 따라서 현재 데이터셋은 `UNDECIDED` 기반 실패, `predicted != expected` 기반 실패, `size_mismatch` 기반 실패를 모두 재현할 수 있다.
- 반대로 `+` expected 와 `cross` 필터 키는 내부에서 모두 `Cross`로 정규화되기 때문에 라벨 차이만으로 인한 오판정은 발생하지 않는다.
- JSON 스키마가 깨졌거나 크기가 맞지 않는 경우에도 프로그램이 죽지 않고 `schema_error`, `size_mismatch`, `missing_filter`, `invalid_label` 같은 사유를 남긴다.
- 콘솔과 JSON 출력에는 참고용으로 MAC 점수와 절대오차 값을 모두 보여 주지만, 최종 판정은 항상 절대오차 기준으로 내린다.
- MAC 연산의 시간 복잡도는 O(N²)이며, 이유는 N x N 행렬의 모든 위치를 한 번씩 방문해 곱셈과 덧셈을 수행하기 때문이다.
- 따라서 3x3은 9회, 5x5는 25회, 13x13은 169회, 25x25는 625회의 위치별 연산이 필요하다.
- 성능 표는 이 연산 횟수 증가와 함께 평균 시간이 커지는 경향을 보여주도록 구성했다.
- 실제 측정값은 실행 환경에 따라 조금씩 달라질 수 있지만, 같은 코드에서 N 이 커질수록 처리 시간이 늘어나는 방향성은 유지된다.
- 보너스 과제인 1차원 배열 최적화나 패턴 생성기는 구현 범위에서 제외했고, 현재 결과는 기본 요구사항 충족에 집중했다.
